# Replit AI Agent Prompt: Master Analysis Upgrade for Sequential Revenue Friction Analysis

## PROJECT CONTEXT

This is the SimpleSequence "Sequential Revenue Friction Analysis" — a free 11-step assessment tool for service business owners. After completing the assessment, users see a `/results` page and can download a PDF report.

**Current architecture:**
- Frontend scoring engine: `client/src/lib/scoring.ts` — runs all pillar calculations, revenue gap math, tier recommendation, and action plan generation client-side. Output stored in `sessionStorage`, user redirected to `/results`.
- Backend mirror: `shared/scoring.ts` — identical scoring logic used exclusively by the server-side PDF generator (`server/pdf-generator.ts`).
- Database: PostgreSQL stores raw form responses. Scores are always re-calculated on demand.
- The `/results` page renders entirely from client-side scoring output.
- The PDF is generated server-side by re-running scoring from stored lead data.

**CRITICAL: Any changes to scoring logic or new calculation functions MUST be updated in BOTH `client/src/lib/scoring.ts` AND `shared/scoring.ts` to keep them in sync. The PDF generator in `server/pdf-generator.ts` must also be updated to render the new sections.**

---

## WHAT WE'RE BUILDING

We're upgrading the `/results` page and PDF report from a basic diagnostic summary into a full "Master Analysis" — a financial narrative that shows the prospect not just where they're leaking revenue, but what closing the gap looks like over time, what inaction costs daily, and how results compound. All new data is derived from existing assessment inputs (no new form fields).

**Brand rules (non-negotiable):**
- Voice: Calm, diagnostic, specific. Never hype ("game-changer," "revolutionary," "cutting-edge"). No exclamation points.
- Framework language: Sequential Revenue, Capture / Convert / Compound, Speed-to-Lead, DBR, Found Money.
- Structure: Problem → Cost → Solution → Outcome.
- REMOVE ALL PRICING from the results page and PDF. No dollar amounts for tiers. Replace with CTAs to book a strategy call. Pricing is discussed on calls only, never in written content.
- Brand colors: Black `#000000`, Cyan/Blue `#1ab1d9`, White `#ffffff`.

### 1.1 Financial Narrative Calculations

```typescript
interface FinancialNarrative {
  // Existing (already calculated)
  currentMonthlyRevenue: number;      // avgJobValue * monthlyJobVolume
  totalMonthlyGap: number;            // captureGap + conversionGap + compoundingGap
  captureGap: number;
  conversionGap: number;
  compoundingGap: number;

  // NEW — Scenario Modeling
  conservativeMonthlyRecovery: number; // totalMonthlyGap * 0.50
  fullMonthlyRecovery: number;         // totalMonthlyGap * 1.00
  conservativeAnnualGain: number;      // conservativeMonthlyRecovery * 12
  fullAnnualGain: number;              // fullMonthlyRecovery * 12

  // NEW — Revenue Comparison
  currentAnnualRevenue: number;        // currentMonthlyRevenue * 12
  potentialAnnualRevenue: number;      // currentAnnualRevenue + conservativeAnnualGain

  // NEW — Cost of Inaction
  dailyCostOfInaction: number;         // totalMonthlyGap / 30 (rounded to nearest dollar)
  sixMonthInactionCost: number;        // totalMonthlyGap * 6
  twelveMonthInactionCost: number;     // totalMonthlyGap * 12
  threeYearInactionCost: number;       // totalMonthlyGap * 36

  // NEW — 12-Month Cumulative Gain (conservative)
  cumulativeGainByMonth: number[];     // Array of 12 values: [month1Total, month2Total, ... month12Total]
                                       // where monthN = conservativeMonthlyRecovery * N

  // NEW — Long-Term Projections
  yearOneGain: number;                 // conservativeAnnualGain (conservative — 50% gap)
  yearThreeGain: number;               // yearOneGain + (fullAnnualGain * 2)  (year 1 conservative, years 2-3 full gap)
  yearFiveGain: number;                // yearOneGain + (fullAnnualGain * 4)  (year 1 conservative, years 2-5 full gap)
}
```
