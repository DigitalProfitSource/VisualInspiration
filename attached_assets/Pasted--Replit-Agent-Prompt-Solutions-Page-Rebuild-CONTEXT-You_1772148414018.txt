## IMPORTANT IMPLEMENTATION NOTES

1. **Preserve the existing design system exactly.** Match colors, fonts, spacing, card styles, background treatments, and responsive behavior from the current site. This is a content restructure, not a redesign.

2. **The "Under the Hood" sections must be collapsible.** Default state = collapsed, showing only the "Under the Hood →" label. On click, it expands to reveal the technical deliverable list with a smooth animation. This keeps the page clean for skimmers while giving detail-seekers what they want.

3. **The "Old Way / New Reality" blocks need clear visual contrast.** Consider: muted/dim text or a subtle red-ish tint for "Old Way", and brighter/teal-accented text for "New Reality". Or use left-right columns with distinct background shading. The goal is the prospect instantly feels the contrast without reading a word.

4. **The loop visual (Section 2) should be responsive.** On mobile, it can stack vertically with arrows pointing down, and the return arrow wrapping from bottom back to top.

5. **Smooth scroll behavior.** If the page is long, consider anchor links from the nav or a sticky sub-nav within the page for "Capture | Convert | Compound" — but only if the site already uses this pattern elsewhere.

6. **Do not create separate CSS/JS files if the current site uses a single-file or component-based architecture.** Match whatever pattern the codebase uses.

7. **Test that the page renders correctly on mobile.** The alternating left-right layout of the pillars should stack cleanly on small screens.
